# Инструкция по стабилизации проекта OnboardPro

Данный документ содержит инструкции по применению исправлений, направленных на улучшение стабильности проекта.

## Обнаруженные проблемы и их решения

### 1. Библиотека `@headlessui/react` отсутствует в контейнере frontend

Хотя библиотека указана в package.json, она не была правильно установлена в контейнере.

**Решение**:

```bash
docker exec -it onboardpro-frontend-1 npm install @headlessui/react
docker restart onboardpro-frontend-1
```

### 2. Проблема с миграциями Alembic

Ошибка в идентификаторах миграций: использование дефисов '-' вместо подчеркиваний '\_'.

**Решение**:

```bash
python backend/fix_migrations.py
```

### 3. Отсутствие переменных окружения

Создан файл `.env` для хранения переменных окружения.

**Решение**:
Заполните файл `.env` своими значениями:

```
# Безопасность
SECRET_KEY=ВашНадежныйКлюч
TELEGRAM_BOT_TOKEN=ваш_токен_бота_telegram
WORKABLE_API_KEY=ваш_ключ_api_workable
GOOGLE_CREDENTIALS_PATH=/путь/к/google_credentials.json
```

### 4. Несовместимость версий bcrypt и passlib

**Решение**:
Понижена версия bcrypt до 4.0.1 для обеспечения совместимости с passlib 1.7.4.

## Применение исправлений

### Шаг 1: Остановите все контейнеры

```bash
docker-compose down
```

### Шаг 2: Примените исправления к миграциям

```bash
cd /Users/magna_mentes/Desktop/Projects/OnboardPro
python backend/fix_migrations.py
```

### Шаг 3: Заполните файл .env своими значениями

Редактируйте файл `.env` и укажите подходящие значения для переменных окружения.

### Шаг 4: Пересоберите и запустите контейнеры

```bash
docker-compose build
docker-compose up -d
```

### Шаг 5: Проверьте работу приложения

Откройте в браузере адрес http://localhost для проверки работоспособности приложения.

## Дополнительные рекомендации

1. Регулярно обновляйте зависимости проекта
2. Добавьте автоматизированные тесты для выявления проблем с миграциями
3. Настройте CI/CD для автоматической проверки совместимости зависимостей
4. Храните переменные окружения в безопасном месте и не включайте их в репозиторий (добавьте `.env` в `.gitignore`)
