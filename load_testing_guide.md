# Руководство по нагрузочному тестированию OnboardPro

## Обзор

Этот документ содержит инструкции по проведению нагрузочного тестирования приложения OnboardPro с использованием Docker и инструмента Locust. Нагрузочное тестирование позволяет оценить производительность системы под различными уровнями нагрузки и выявить потенциальные узкие места.

## Подготовка к тестированию

### Предварительные требования

- Docker и Docker Compose установлены на вашей системе
- Клонированный репозиторий OnboardPro

### Файлы для нагрузочного тестирования

В репозитории уже подготовлены следующие файлы для нагрузочного тестирования:

1. `backend/load_test.py` - скрипт Locust для симуляции пользовательской нагрузки
2. `backend/Dockerfile.loadtest` - Dockerfile для создания контейнера с Locust
3. `docker-compose.loadtest.yml` - конфигурация Docker Compose для запуска тестирования
4. `backend/analyze_load_test.py` - скрипт для анализа результатов тестирования

## Запуск нагрузочного тестирования

### Шаг 1: Запуск контейнеров

```bash
# Запустите контейнеры с использованием специальной конфигурации для нагрузочного тестирования
docker-compose -f docker-compose.loadtest.yml up -d
```

Эта команда запустит три контейнера:

- `backend` - бэкенд приложения на FastAPI
- `frontend` - фронтенд приложения на React
- `loadtest` - контейнер с Locust для нагрузочного тестирования

### Шаг 2: Доступ к веб-интерфейсу Locust

Откройте в браузере адрес [http://localhost:8089](http://localhost:8089) для доступа к веб-интерфейсу Locust.

### Шаг 3: Настройка и запуск теста

В веб-интерфейсе Locust:

1. Укажите количество пользователей (Number of users) - рекомендуется начать с 10-20
2. Укажите скорость появления пользователей (Spawn rate) - рекомендуется 1-2 пользователя в секунду
3. Нажмите кнопку "Start swarming" для начала тестирования

### Шаг 4: Мониторинг тестирования

Во время тестирования вы можете наблюдать в реальном времени:

- Графики RPS (запросов в секунду)
- Время отклика
- Количество успешных и неудачных запросов
- Статистику по каждому эндпоинту API

### Шаг 5: Экспорт результатов

После завершения тестирования (рекомендуется запускать тест на 5-10 минут):

1. Нажмите кнопку "Stop" для остановки теста
2. Перейдите на вкладку "Download Data"
3. Скачайте результаты в формате JSON, нажав на кнопку "Download Report" в разделе JSON

## Анализ результатов

### Использование скрипта анализа

```bash
# Скопируйте файл с результатами из хост-системы в контейнер
docker cp ./results.json onboardpro_backend_1:/app/

# Запустите скрипт анализа внутри контейнера
docker exec onboardpro_backend_1 python analyze_load_test.py /app/results.json

# Скопируйте сгенерированный график из контейнера
docker cp onboardpro_backend_1:/app/load_test_results.png ./
```

### Интерпретация результатов

После выполнения скрипта анализа вы получите:

1. Текстовый отчет с основными метриками производительности
2. Информацию о самых медленных эндпоинтах
3. Информацию об эндпоинтах с наибольшим процентом ошибок
4. Графики времени отклика и RPS для каждого эндпоинта
5. Рекомендации по оптимизации на основе результатов

## Сценарии тестирования

Скрипт `load_test.py` симулирует следующие действия пользователей:

1. Авторизация пользователя (HR)
2. Получение списка пользователей
3. Получение списка задач
4. Получение списка планов онбординга
5. Создание новых задач
6. Обновление статуса задач
7. Создание и получение отзывов

Эти сценарии охватывают основные функции приложения и позволяют оценить производительность системы в целом.

## Рекомендации по проведению тестирования

1. **Постепенное увеличение нагрузки**: Начните с небольшого количества пользователей (10-20) и постепенно увеличивайте до достижения предела производительности системы.

2. **Длительное тестирование**: Для выявления проблем с утечкой памяти или деградацией производительности со временем, проводите длительные тесты (30+ минут).

3. **Тестирование отдельных эндпоинтов**: Для более детального анализа можно модифицировать скрипт `load_test.py`, чтобы сосредоточиться на конкретных эндпоинтах.

4. **Мониторинг ресурсов**: Во время тестирования рекомендуется мониторить использование CPU, памяти и дисковых операций контейнеров с помощью команды `docker stats`.

## Устранение неполадок

### Проблема: Контейнер с Locust не запускается

**Решение**: Проверьте логи контейнера:

```bash
docker logs onboardpro_loadtest_1
```

### Проблема: Не удается подключиться к бэкенду

**Решение**: Убедитесь, что контейнер с бэкендом запущен и проходит проверку здоровья:

```bash
docker ps
docker logs onboardpro_backend_1
```

### Проблема: Высокий процент ошибок в тестах

**Решение**: Проверьте логи бэкенда для выявления причин ошибок и убедитесь, что база данных правильно инициализирована:

```bash
docker exec onboardpro_backend_1 python seed_data.py
```

## Заключение

Нагрузочное тестирование является важной частью оценки качества приложения OnboardPro. Регулярное проведение таких тестов позволяет своевременно выявлять проблемы производительности и принимать меры по их устранению до того, как они повлияют на работу пользователей в реальных условиях.
