### Спринт 7 — Синхронизация

## Архитектура

### Структура бэкенда

#### Модели данных (/backend/models.py)

- **User**: модель пользователя с ролями (employee, manager, hr), полями для ФИО, почты, пароля, телефона и отдела
- **OnboardingPlan**: планы адаптации с указанием роли, названия и описания
- **TaskTemplate**: шаблоны задач с приоритетом, продолжительностью, ролью и отделом
- **Task**: задачи, связанные с планами и шаблонами, с полями для статуса, дедлайна и приоритета
- **Feedback**: модель для обратной связи между пользователями
- **Analytics**: хранение аналитических данных

#### Эндпоинты API (/backend/api/\*)

- **/api/plans**: CRUD-операции над планами адаптации (GET, POST, PUT, DELETE)
- **/api/task_templates**: CRUD-операции над шаблонами задач (GET, POST, PUT, DELETE)
- **/api/tasks**: управление задачами (создание, обновление, изменение статуса)
- **/api/analytics**: получение аналитических данных о задачах и пользователях
- **/api/portal**: API для портала кандидатов

#### Зависимости

- FastAPI для API
- SQLAlchemy для ORM
- SQLite для хранения данных
- JWT для аутентификации

### Структура фронтенда

#### Страницы (/frontend/src/pages/\*)

- **Dashboard.jsx**: основной дашборд для сотрудников с задачами
- **ManagerDashboard.jsx**: дашборд для менеджеров
- **HRDashboard.jsx**: дашборд для HR с аналитикой и KPI
- **Plans.jsx**: страница для управления планами адаптации
- **TaskTemplates.jsx**: страница для управления шаблонами задач
- **Login.jsx**: страница авторизации
- **Feedback.jsx**: система обратной связи
- **Profiles.jsx**: управление профилями пользователей
- **Integrations.jsx**: настройка интеграций (Google Calendar и др.)

#### Компоненты (/frontend/src/components/\*)

- **Layout.js**: основной макет с навигацией
- **ProtectedRoute.jsx**: компонент защиты роутов по ролям
- **Modal.jsx**, **ConfirmDialog.jsx**: UI-компоненты для модальных окон
- **TaskModal.jsx**: форма создания/редактирования задач
- **PlanForm.jsx**: форма для планов адаптации (вызывается из Plans.jsx)
- **TaskTemplateForm.jsx**: форма для шаблонов задач

#### Роутинг (/frontend/src/App.js)

- Реализовано с использованием React Router
- Роуты защищены по ролям через компонент ProtectedRoute
- Навигационное меню зависит от роли пользователя

### Авторизация

#### Роли пользователей

- **HR**: полный доступ ко всем функциям
- **Manager**: доступ к дашбордам, управлению командой, ограниченному просмотру планов
- **Employee**: доступ только к личному дашборду и обратной связи
- **Candidate**: доступ к порталу кандидатов

#### Механизм авторизации (/backend/auth.py)

- Основан на JWT-токенах
- Токены хранятся в localStorage браузера
- Реализована защита роутов в зависимости от роли

## Функционал

### Шаблоны задач

#### API (/backend/api/task_templates.py)

- GET /api/task_templates: получение всех шаблонов
- POST /api/task_templates: создание нового шаблона (только HR)
- PUT /api/task_templates/{id}: обновление шаблона (только HR)
- DELETE /api/task_templates/{id}: удаление шаблона (только HR)

#### UI (/frontend/src/pages/TaskTemplates.jsx)

- Таблица для отображения существующих шаблонов
- Модальное окно для создания/редактирования шаблона
- Фильтрация по категориям
- Подтверждение перед удалением

### Планы адаптации

#### API (/backend/api/plans.py)

- GET /api/plans: получение всех планов
- POST /api/plans: создание нового плана (только HR)
- PUT /api/plans/{id}: обновление плана (только HR)
- DELETE /api/plans/{id}: удаление плана (только HR)

#### UI (/frontend/src/pages/Plans.jsx)

- Компонент для отображения планов адаптации
- Форма для создания/редактирования планов
- Возможность добавлять шаблонные задачи при создании плана
- Возможность назначения плана сотруднику

### Портал кандидатов

- API для работы с кандидатами (/backend/api/portal.py)
- Специализированный интерфейс для кандидатов
- Отдельные роуты и авторизация для кандидатов

### Производительность

- Кэширование шаблонов задач (реализовано в task_templates.py)
- Пагинация результатов API
- Оптимизированные SQL-запросы с использованием индексов
- Составные индексы для часто используемых запросов

### Безопасность

- Ролевые проверки на всех защищенных эндпоинтах
- Защита роутов на фронтенде через ProtectedRoute компонент
- Хеширование паролей
- Валидация данных с помощью Pydantic на бэкенде

### Интеграции

- Интеграция с Google Calendar для синхронизации задач
- API для внешних интеграций

## UI

### Блок "Управление планами адаптации"

#### Проблема интеграции

**Выявленная несогласованность**: Хотя страницы Plans.jsx и TaskTemplates.jsx реализованы с полным функционалом для управления планами адаптации и шаблонами, **они не интегрированы в систему навигации приложения**:

1. В файле App.js отсутствуют маршруты к /plans и /templates
2. В навигационном меню Layout.js отсутствуют ссылки на эти страницы
3. Согласно user_guide_ru.md, доступ к этим страницам должен осуществляться через вкладку "Планы адаптации" на страницах Manager Dashboard или HR Dashboard, но эта функциональность не реализована

#### Расположение компонентов

- Компонент для работы с планами: /frontend/src/pages/Plans.jsx
- Компонент для работы с шаблонами: /frontend/src/pages/TaskTemplates.jsx
- Формы для редактирования: /frontend/src/components/specific/PlanForm.jsx и TaskTemplateForm.jsx

### Кнопки/меню для шаблонов и планов

- В HR Dashboard отсутствуют явные элементы для перехода к страницам управления планами и шаблонами
- Функциональность реализована, но не интегрирована в интерфейс приложения
- Требуется добавление соответствующих пунктов меню или вкладок в HR Dashboard

### Окружение

- Docker для контейнеризации

### Проблемы

- **Основная проблема UI**: Отсутствие интеграции страниц управления планами и шаблонами в систему навигации
- **Проблема с авторизацией**: Корректно реализованы проверки ролей, но в UI отсутствуют пути навигации к защищенным страницам
- **Несогласованность документации и реализации**: Руководство пользователя описывает функциональность, которая реализована, но не доступна через интерфейс

## Причина отсутствия UI для шаблонов

Основная причина недоступности UI для управления шаблонами и планами адаптации:

1. **Отсутствие маршрутов**: В App.js не добавлены маршруты к страницам Plans.jsx и TaskTemplates.jsx
2. **Отсутствие пунктов навигации**: В Layout.js отсутствуют ссылки на эти страницы
3. **Незавершенная интеграция**: Не реализован интерфейс для перехода к этим страницам из HR Dashboard

## Решение проблемы UI для планов адаптации и шаблонов (06.05.2025)

### Реализованное решение

Вместо создания отдельных страниц и маршрутов для управления планами адаптации и шаблонами задач, была выбрана стратегия интеграции всей функциональности непосредственно в страницу Manager Dashboard, что соответствует описанию в руководстве пользователя.

#### Реализация интерфейса в Manager Dashboard

Раздел "Управление планами адаптации" в ManagerDashboard.jsx был полностью переработан и теперь включает:

1. **Блок управления шаблонами задач (только для HR)**:

   - Отображение списка шаблонов с возможностью свернуть/развернуть
   - Кнопка для создания новых шаблонов
   - Функции для редактирования и удаления шаблонов
   - Отображение основной информации о шаблоне (название, описание, роль, отдел)

2. **Блок управления планами адаптации**:

   - Сетка карточек планов с адаптивной компоновкой для разных размеров экрана
   - Визуализация прогресса выполнения задач в каждом плане
   - Информация о назначенном сотруднике для каждого плана
   - Кнопки для создания, редактирования и удаления планов

3. **Модальные окна интеграции**:
   - При создании/редактировании плана используется компонент PlanForm.jsx
   - При создании/редактировании шаблона используется компонент TaskTemplateForm.jsx
   - Диалоги подтверждения для операций удаления

#### Преимущества данного подхода

1. **Консолидация функциональности**: Весь функционал управления планами и шаблонами доступен в одном месте
2. **Соответствие руководству пользователя**: Реализация полностью соответствует описанию в документации
3. **Простота доступа**: Нет необходимости переходить между разными страницами
4. **Интуитивный интерфейс**: Визуальное разделение на блоки шаблонов и планов
5. **Управление правами доступа**: Функции создания/редактирования/удаления отображаются только для пользователей с ролью HR

### Интеграция с существующими компонентами

1. **PlanForm.jsx и TaskTemplateForm.jsx**:

   - Используются существующие компоненты форм без модификации
   - Вызываются через модальные окна непосредственно в ManagerDashboard.jsx

2. **Модальные окна**:

   - Используется универсальный компонент Modal.jsx для создания диалогов
   - Реализованы диалоги подтверждения удаления с информацией о удаляемых объектах

3. **API интеграция**:
   - Реализованы функции для взаимодействия с API бэкенда
   - Добавлена обработка ошибок и уведомления через toast-сообщения

### Обновление документации

1. **Руководство пользователя**:
   - Добавлены подробные инструкции по использованию нового интерфейса
   - Указаны особенности доступа к функциям в зависимости от роли пользователя
   - Добавлены пошаговые инструкции для создания, редактирования и удаления планов и шаблонов

### Дополнительные улучшения

1. **Визуализация данных**:

   - Отображение индикаторов прогресса выполнения задач для каждого плана
   - Цветовая индикация различных типов объектов (планы, шаблоны)
   - Адаптивная компоновка для различных размеров экрана

2. **UX-улучшения**:
   - Уведомления о результатах операций через toast-сообщения
   - Кнопка обновления для синхронизации данных с сервером
   - Индикаторы загрузки при выполнении асинхронных операций

### Выводы

Реализованное решение полностью устраняет проблему отсутствия UI для управления планами адаптации и шаблонами задач без необходимости создания отдельных страниц и маршрутов. Данный подход обеспечивает простоту использования и соответствует изначальному дизайну системы, описанному в документации.
