## Архитектура системы
OnboardPro следует современной архитектуре веб-приложения:

- **Backend**: FastAPI (Python) с базой данных SQLite
- **Frontend**: React с Tailwind CSS (PostCSS 8), адаптивным дизайном и Heroicons
- **Инфраструктура**: Docker и docker-compose для контейнеризации
- **Аутентификация**: JWT-based система аутентификации

## Среда разработки

### Предварительные требования
- Docker и Docker Compose
- Node.js (v16+) и npm
- Python 3.9+

### Инструкции по настройке
1. Клонировать репозиторий
2. Запустить `docker-compose up -d` для старта контейнеров
3. Открыть приложение по адресу http://localhost:3000
4. Запустить `./frontend/validate_setup.sh` для проверки настройки

### Переменные окружения
- `DATABASE_URL`: Путь к базе данных SQLite
- `SECRET_KEY`: Секретный ключ для JWT
- `FRONTEND_URL`: URL фронтенд-приложения

## Backend API

### Эндпоинты аутентификации
- `POST /auth/login`: Вход пользователя
- `POST /auth/refresh`: Обновление JWT токена
- `GET /users/me`: Получение информации о текущем пользователе

### Управление пользователями
- `GET /users`: Список всех пользователей (только для HR)
- `POST /users`: Создание нового пользователя (только для HR)
- `PUT /users/{id}`: Обновление пользователя (только для HR)
- `DELETE /users/{id}`: Удаление пользователя (только для HR)

### Управление задачами
- `GET /tasks`: Список задач
- `POST /tasks`: Создание задачи
- `PUT /tasks/{id}`: Обновление задачи
- `DELETE /tasks/{id}`: Удаление задачи

### Обратная связь
- `GET /feedback`: Список обратной связи
- `POST /feedback`: Создание обратной связи
- `PUT /feedback/{id}`: Обновление обратной связи
- `DELETE /feedback/{id}`: Удаление обратной связи

## Frontend реализация

### Структура компонентов
- `App.js`: Главный компонент приложения
- `components/`: Переиспользуемые UI компоненты
- `pages/`: Компоненты страниц
- `context/`: React контекст провайдеры
- `hooks/`: Пользовательские React хуки
- `utils/`: Утилитные функции

### Адаптивный дизайн
- Десктоп (≥1280px): Полная навигация с иконками и текстом
- Планшет (≥768px, <1280px): Навигация только с иконками
- Мобильный (<768px): Навигация с бургер-меню

### Управление состоянием
- React Context API для глобального состояния
- Локальное состояние компонентов для UI элементов

## Схема базы данных

### Таблица Users
- `id`: Первичный ключ
- `email`: Email пользователя (уникальный)
- `password_hash`: Хеш пароля
- `role`: Роль пользователя (HR, Manager, Employee)
- `department`: Отдел пользователя
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

### Таблица Tasks
- `id`: Первичный ключ
- `title`: Название задачи
- `description`: Описание задачи
- `assigned_to`: ID пользователя (внешний ключ)
- `assigned_by`: ID пользователя (внешний ключ)
- `status`: Статус задачи
- `due_date`: Срок выполнения
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

### Таблица Feedback
- `id`: Первичный ключ
- `content`: Содержание обратной связи
- `from_user`: ID пользователя (внешний ключ)
- `to_user`: ID пользователя (внешний ключ)
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

## Развертывание

### Развертывание в Docker
1. Собрать Docker образы: `docker-compose build`
2. Запустить контейнеры: `docker-compose up -d`
3. Открыть приложение по адресу http://localhost:3000

### Продакшн развертывание
1. Настроить продакшн сервер
2. Настроить переменные окружения
3. Собрать и развернуть Docker контейнеры
4. Настроить обратный прокси (рекомендуется Nginx)
5. Настроить SSL сертификаты

## Безопасность

### Аутентификация
- JWT токены с истечением срока действия
- Хеширование паролей с bcrypt
- Контроль доступа на основе ролей

### Защита данных
- Валидация входных данных
- Предотвращение SQL-инъекций
- Защита от XSS
- Защита от CSRF

### Безопасность API
- Ограничение скорости запросов
- Валидация запросов
- Обработка ошибок

## Устранение неполадок

### Распространенные проблемы
- **Подключение к базе данных**: Проверить переменную окружения DATABASE_URL
- **Аутентификация**: Проверить секретный ключ JWT
- **Сборка фронтенда**: Проверить npm зависимости
- **Проблемы с Docker**: Проверить установку Docker и docker-compose

### Логи
- Логи бэкенда: `docker-compose logs backend`
- Логи фронтенда: `docker-compose logs frontend`
- Логи базы данных: `docker-compose logs db`
