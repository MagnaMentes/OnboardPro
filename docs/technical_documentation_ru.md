## Архитектура системы
OnboardPro следует современной архитектуре веб-приложения:

- **Backend**: FastAPI (Python) с базой данных SQLite
- **Frontend**: React с Tailwind CSS и Heroicons
- **Инфраструктура**: Docker и docker-compose для контейнеризации
- **Аутентификация**: JWT-based система аутентификации
- **Внешние интеграции**: Telegram Bot, Google Calendar, Workable API

## Среда разработки

### Предварительные требования
- Docker и Docker Compose
- Node.js (v18+) и npm
- Python 3.11+

### Инструкции по установке
1. Клонировать репозиторий
2. Создать файл `.env` с необходимыми переменными окружения
3. Запустить контейнеры: `docker-compose up -d`
4. Открыть приложение по адресу http://localhost:3000

### Переменные окружения
Необходимые переменные окружения:
- `DATABASE_URL`: Путь к базе данных SQLite (по умолчанию: sqlite:///onboardpro.db)
- `SECRET_KEY`: Секретный ключ JWT для аутентификации
- `TELEGRAM_BOT_TOKEN`: Токен Telegram Bot API
- `GOOGLE_CREDENTIALS_PATH`: Путь к файлу учетных данных Google API
- `WORKABLE_API_KEY`: Ключ API Workable для интеграции с рекрутингом

Переменные окружения Frontend:
- `REACT_APP_API_URL`: URL Backend API (по умолчанию: http://localhost:8000)

## Backend API

### Эндпоинты аутентификации
- `POST /auth/login`: Вход пользователя
- `POST /auth/refresh`: Обновление JWT токена
- `GET /users/me`: Получение информации о текущем пользователе

### Управление пользователями
- `GET /users`: Список всех пользователей (только для HR)
- `POST /users`: Создание нового пользователя (только для HR)
- `PUT /users/{id}`: Обновление пользователя (только для HR)
- `DELETE /users/{id}`: Удаление пользователя (только для HR)

### Управление задачами
- `GET /tasks`: Список задач
- `POST /tasks`: Создание задачи
- `PUT /tasks/{id}`: Обновление задачи
- `DELETE /tasks/{id}`: Удаление задачи

### Управление планами
- `GET /plans`: Список планов адаптации
- `POST /plans`: Создание нового плана
- `PUT /plans/{id}`: Обновление плана
- `DELETE /plans/{id}`: Удаление плана

### Обратная связь
- `GET /feedback`: Список отзывов
- `POST /feedback`: Создание отзыва
- `PUT /feedback/{id}`: Обновление отзыва
- `DELETE /feedback/{id}`: Удаление отзыва

## Frontend реализация

### Структура компонентов
- `App.js`: Главный компонент приложения
- `components/`: Переиспользуемые UI компоненты
- `pages/`: Компоненты страниц
  - `ManagerDashboard.js`: Интерфейс менеджера для управления задачами и планами
- `context/`: React контекст провайдеры
- `hooks/`: Пользовательские React хуки
- `utils/`: Вспомогательные функции

### Адаптивный дизайн
- Десктоп (≥1280px): Полная навигация с иконками и текстом
- Планшет (≥768px, <1280px): Навигация только с иконками
- Мобильный (<768px): Навигация через бургер-меню

### Управление состоянием
- React Context API для глобального состояния
- Локальное состояние компонентов для UI элементов
- Управление состоянием форм с контролируемыми компонентами

## Схема базы данных

### Таблица Users
- `id`: Первичный ключ
- `email`: Email пользователя (уникальный)
- `password_hash`: Хеш пароля
- `role`: Роль пользователя (HR, Manager, Employee)
- `department`: Отдел пользователя
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

### Таблица Tasks
- `id`: Первичный ключ
- `title`: Название задачи
- `description`: Описание задачи
- `user_id`: ID назначенного пользователя (внешний ключ)
- `plan_id`: ID плана (внешний ключ)
- `priority`: Приоритет задачи (low, medium, high)
- `status`: Статус задачи
- `deadline`: Срок выполнения
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

### Таблица Plans
- `id`: Первичный ключ
- `title`: Название плана
- `description`: Описание плана
- `role`: Целевая роль (employee, manager)
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

### Таблица Feedback
- `id`: Первичный ключ
- `content`: Содержание отзыва
- `from_user`: ID пользователя-отправителя (внешний ключ)
- `to_user`: ID пользователя-получателя (внешний ключ)
- `created_at`: Временная метка создания
- `updated_at`: Временная метка обновления

## Развертывание

### Docker развертывание
1. Создать файл `.env` с необходимыми переменными окружения
2. Собрать Docker образы: `docker-compose build`
3. Запустить контейнеры: `docker-compose up -d`
4. Открыть приложение по адресу http://localhost:3000

### Конфигурация контейнеров
- Backend контейнер:
  - Эндпоинт проверки здоровья: `/health`
  - Политика автоматического перезапуска: `unless-stopped`
  - Монтирование томов для разработки
- Frontend контейнер:
  - Сервер разработки на порту 3000
  - Политика автоматического перезапуска: `unless-stopped`
  - Монтирование томов для разработки
  - Том node_modules для производительности

### Продакшн развертывание
1. Настроить продакшн сервер
2. Настроить переменные окружения
3. Собрать и развернуть Docker контейнеры
4. Настроить обратный прокси (рекомендуется Nginx)
5. Настроить SSL сертификаты

## Безопасность

### Аутентификация
- JWT токены с истечением срока действия
- Хеширование паролей с bcrypt
- Контроль доступа на основе ролей

### Защита данных
- Валидация входных данных
- Защита от SQL-инъекций
- Защита от XSS
- Защита от CSRF

### Безопасность API
- Ограничение частоты запросов
- Валидация запросов
- Обработка ошибок
- Защита переменных окружения

## Устранение неполадок

### Распространенные проблемы
- **Переменные окружения**: Убедитесь, что все необходимые переменные установлены в `.env`
- **Подключение к базе данных**: Проверьте переменную окружения DATABASE_URL
- **Аутентификация**: Проверьте секретный ключ JWT
- **Сборка Frontend**: Проверьте зависимости npm
- **Проблемы с Docker**: Проверьте установку Docker и docker-compose

### Логи
- Логи Backend: `docker-compose logs backend`
- Логи Frontend: `docker-compose logs frontend`
