# 🎨 Фронтенд OnboardPro

## 🛠 Технологии

- 📄 HTML5
- 💅 TailwindCSS для стилизации с PostCSS 8
- ⚡️ React для интерактивности
- 🔤 Inter font для типографики
- 📱 Адаптивный дизайн для всех устройств

## 📁 Структура проекта

```
frontend/
├── src/                # 📂 Исходный код
│   ├── components/     # 🧩 Компоненты
│   │   └── Layout.js   # 📑 Базовый шаблон
│   ├── pages/          # 📄 Страницы
│   │   ├── Dashboard.js # 📊 Панель управления
│   │   ├── Login.js    # 🔑 Страница входа
│   │   └── ...         # 🔐 Другие страницы
│   ├── App.js          # 🧠 Основной компонент
│   ├── index.css       # 🎨 Основной файл стилей с Tailwind
│   └── index.js        # 🚀 Точка входа
├── build/              # 📦 Скомпилированные файлы
├── package.json        # 📋 Зависимости и скрипты
└── tailwind.config.js  # ⚙️ Конфигурация TailwindCSS с адаптивными брейкпоинтами
```

## ⚙️ Настройка и установка

1. 📥 Установите Node.js и npm
2. 🔧 Выполните:
   ```bash
   npm install --legacy-peer-deps
   ```
3. 🚀 Для разработки запустите:
   ```bash
   npm start
   ```
4. 📦 Для сборки:
   ```bash
   npm run build
   ```

## 🧱 Компоненты

### 📑 Базовый шаблон (Layout.js)

Содержит:

- 🗂 Header с навигацией
- 📄 Основной контейнер для контента
- 📝 Footer с копирайтом и социальными ссылками

### 🔑 Форма входа (Login.js)

- 📱 Адаптивный дизайн с поддержкой мобильных устройств
- ✅ Валидация на стороне клиента
- ⚠️ Обработка ошибок
- 🔒 Безопасная аутентификация

## ⚡️ React компоненты

### 🔐 Аутентификация (Login.js)

- 📤 Отправка формы через fetch API
- 💾 Сохранение JWT в localStorage
- 🔄 Обработка ошибок и перенаправление
- 🔒 Защищенные маршруты

### 🧭 Навигация (Layout.js)

- 🏗 Динамическая отрисовка меню на основе роли пользователя
- 🔄 Проверка авторизации
- 🚪 Функционал выхода из системы
- 🎯 Подсветка активного пункта меню
- 📱 Адаптивное меню для мобильных устройств

## 📋 Рекомендации по разработке

1. 🎯 Следуйте методологии Utility-First CSS от TailwindCSS
2. 📝 Используйте компонентный подход React
3. 📱 Придерживайтесь принципа Mobile First с использованием адаптивных брейкпоинтов
4. 🌐 Тестируйте в разных браузерах
5. ♿️ Обеспечивайте доступность (a11y) для всех пользователей
