## Основные принципы

В проекте OnboardPro реализованы следующие принципы проектирования интерфейса:

### 1. Интуитивная понятность

- Визуальное кодирование элементов с использованием цвета, формы и иконок
- Контекстные подсказки для интерактивных элементов
- Последовательность и предсказуемость взаимодействий
- Минимально необходимое количество действий для достижения цели

### 2. Мгновенная обратная связь

- Визуальное подтверждение действий пользователя
- Плавные анимации для обозначения переходов между состояниями
- Индикаторы загрузки и процесса выполнения действий
- Всплывающие уведомления о результатах операций

### 3. Согласованность интерфейса

- Единая система дизайна на основе TailwindCSS
- Стандартизированные компоненты с унифицированным поведением
- Согласованное использование цветов, отступов и типографики
- Предсказуемый пользовательский опыт на всех страницах

### 4. Эффективность работы

- Группировка связанных элементов управления
- Циклическое переключение между часто используемыми статусами
- Сокращение числа кликов для выполнения типовых операций
- Визуальная иерархия и акценты на основных действиях

## Компоненты интерфейса

### Карточки

Карточки используются для представления отдельных сущностей (задач, пользователей, планов). Структура карточки:

- **Заголовок**: отражает основную информацию и имеет цветовое кодирование
- **Тело**: содержит детали и атрибуты сущности
- **Действия**: интерактивные элементы для взаимодействия с сущностью

Примеры реализации:

```jsx
<Card title="Название задачи" priority="high" actions={<CardActions />}>
  <CardContent />
</Card>
```

### Кнопки и действия

Система кнопок следует принципам иерархии важности:

- **Первичные (primary)**: Основные, важные действия

  - Цвет: синий (blue-600)
  - Использование: Подтверждение, сохранение, основные действия

- **Вторичные (secondary)**: Второстепенные действия

  - Цвет: серый (gray-200/300)
  - Использование: Отмена, второстепенные действия

- **Деструктивные (destructive)**: Действия удаления/отмены
  - Цвет: красный (red-600)
  - Использование: Удаление, отмена подписки
  - **Важно**: Все деструктивные действия должны сопровождаться диалогом подтверждения для предотвращения случайного удаления (добавлено 14.05.2025)

### Подтверждение критических операций (добавлено 14.05.2025)

Для любых действий с необратимыми последствиями применяется паттерн подтверждения:

- **Диалог подтверждения**: Модальное окно с чётким заголовком, описанием последствий и кнопками действий
- **Название объекта**: Обязательно указывается имя/заголовок объекта, который будет удален
- **Цветовая индикация**: Кнопка подтверждения выполнена в красном цвете (red-600)
- **Предупреждение**: Текстовое предупреждение о необратимости действия

Пример компонента подтверждения удаления:

```jsx
<Modal
  isOpen={isDeleteModalOpen}
  onClose={() => setIsDeleteModalOpen(false)}
  title="Удаление задачи"
  footer={
    <>
      <button
        type="button"
        onClick={() => setIsDeleteModalOpen(false)}
        className="px-4 py-2 border border-gray-300 rounded-md shadow-sm text-sm font-medium text-gray-700 bg-white hover:bg-gray-50"
      >
        Отмена
      </button>
      <button
        type="button"
        onClick={handleDelete}
        className="ml-3 inline-flex justify-center px-4 py-2 border border-transparent rounded-md shadow-sm text-sm font-medium text-white bg-red-600 hover:bg-red-700"
      >
        Удалить
      </button>
    </>
  }
>
  <p className="text-sm text-gray-500">
    Вы действительно хотите удалить задачу "{objectName}"? Это действие нельзя
    отменить.
  </p>
</Modal>
```

### Модальные окна

Модальные окна используются для фокусировки внимания на конкретной операции:

- **Заголовок**: ясно обозначает назначение окна
- **Содержимое**: форма или информация для взаимодействия
- **Кнопки действий**: располагаются в нижней части
- **Закрытие**: доступно через иконку, клик вне окна и клавишу Esc

Реализация с анимациями появления и исчезновения:

```jsx
<div
  className={`fixed inset-0 bg-black transition-opacity duration-300 z-40 ${
    isOpen ? "opacity-50" : "opacity-0 pointer-events-none"
  }`}
  onClick={onClose}
></div>
<div
  className={`fixed inset-0 flex items-center justify-center z-50 transition-opacity duration-300 ${
    isOpen ? "opacity-100" : "opacity-0 pointer-events-none"
  }`}
>
  <div
    className={`bg-white rounded-lg shadow-xl transform transition-all duration-300 ${
      isOpen ? "scale-100" : "scale-95"
    }`}
  >
    {/* Содержимое модального окна */}
  </div>
</div>
```

### Формы

Формы следуют единому стилю для обеспечения согласованности:

- **Метки**: четкие и размещенные над полями ввода
- **Поля ввода**: визуально выделенные рамки и состояния фокуса
- **Валидация**: мгновенная обратная связь при ошибках ввода
- **Кнопки действий**: визуально различимые с соответствующими цветами

## Управление статусами задач

### Компонент TaskStatusToggle

Компонент для переключения статуса задачи с улучшенным UX:

- **Горизонтальные кнопки** с цветовой кодировкой для каждого статуса
- **Иконки и текст** для улучшения понятности
- **Активное состояние** с визуальным выделением текущего статуса
- **Анимации** при наведении и переключении

## Улучшения UI для операций с задачами (добавлено 14.05.2025)

### Предотвращение случайного удаления

Для предотвращения случайного удаления задач реализован двухэтапный механизм:

- **Модальное окно подтверждения** с информацией о задаче
- **Двойное подтверждение** через кнопки "Отмена" и "Удалить"
- **Единообразное поведение** во всех представлениях: мобильном, планшетном и десктопном

### Улучшенные заголовки колонок

Для повышения визуальной согласованности интерфейса:

- **Иконка вместо текста** в заголовке колонки действий
- **Визуальный контекст** через атрибут `title` для доступности
- **Единообразие между представлениями** с использованием компонента AdjustmentsVerticalIcon

```jsx
<TaskStatusToggle
  status={task.status}
  onChange={handleStatusChange}
  disabled={isLoading}
/>
```

### Компонент TaskStatusTip

Всплывающие подсказки с информацией о статусе и возможных действиях:

- **Контекстная информация** при наведении на элемент
- **Настраиваемое расположение** относительно элемента
- **Визуализация цикла статусов** с показом следующего статуса

```jsx
<TaskStatusTip
  content={<>Текущий статус: {statusLabel}</>}
  position="bottom"
  showStatusCycle={true}
  currentStatus={status}
>
  <StatusIndicator status={status} />
</TaskStatusTip>
```

### Компонент StatusChangeIndicator

Визуальное уведомление о смене статуса задачи:

- **Центральное уведомление** с анимацией появления
- **Цветовое кодирование** в соответствии с новым статусом
- **Анимированное появление и исчезновение**
- **Эмодзи и текст** для быстрого восприятия

```jsx
<StatusChangeIndicator
  show={showIndicator}
  status={newStatus}
  onAnimationComplete={() => setShowIndicator(false)}
/>
```

## Система уведомлений

В приложении реализованы следующие типы уведомлений:

### Toast-уведомления

- **Кратковременные сообщения** об успехе или ошибке операций
- **Автоматическое исчезновение** через установленный промежуток времени
- **Цветовая кодировка** по типам: успех (зеленый), ошибка (красный), информация (синий)

### Контекстные индикаторы

- **Визуальная обратная связь** при действиях пользователя
- **Анимированные переходы** между состояниями
- **Размещение рядом с контекстом** действия

### In-app уведомления

- **Хранение в интерфейсе** для важных сообщений
- **Счетчик непрочитанных** в шапке приложения
- **Группировка** похожих уведомлений

## Анимации и переходы

### Типы анимаций

1. **Микроанимации** для обратной связи:

   - Нажатие кнопок (масштабирование, изменение тени)
   - Изменение статуса (плавная смена цвета и формы)
   - Появление и исчезновение всплывающих элементов

2. **Переходы** для изменения состояний:

   - Появление модальных окон с масштабированием
   - Плавное появление панелей фильтров
   - Смена видов и режимов отображения

3. **Индикаторы процесса**:
   - Анимированные индикаторы загрузки
   - Прогресс-бары для длительных операций
   - Пульсирующие индикаторы для привлечения внимания

### CSS-классы для анимаций

В проекте используются следующие CSS-классы для создания анимаций:

```css
/* Анимация появления элементов */
.fade-in {
  animation: fadeIn 0.3s ease-in-out;
}

@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(-10px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* Анимация пульсации для привлечения внимания */
.pulse {
  animation: pulse 2s infinite;
}

@keyframes pulse {
  0% {
    transform: scale(1);
    opacity: 1;
  }
  50% {
    transform: scale(1.05);
    opacity: 0.8;
  }
  100% {
    transform: scale(1);
    opacity: 1;
  }
}
```

### Переходы состояний

Для плавных переходов между состояниями используются CSS-transition:

```css
.transition-all {
  transition-property: all;
  transition-timing-function: ease-in-out;
  transition-duration: 300ms;
}

.transition-transform {
  transition-property: transform;
  transition-timing-function: ease-in-out;
  transition-duration: 150ms;
}

.transition-opacity {
  transition-property: opacity;
  transition-timing-function: ease-in-out;
  transition-duration: 300ms;
}
```
