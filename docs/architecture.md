# Документация по архитектуре

## Обзор системы

OnboardPro — современное веб-приложение, построенное на Django и React, разработанное для оптимизации процесса онбординга сотрудников. Система следует паттерну микросервисной архитектуры с четким разделением ответственности между backend и frontend компонентами.

## Технологический стек

### Backend
- Django 4.2
- Django REST Framework
- PostgreSQL
- JWT Аутентификация
- Celery (для асинхронных задач)
- Redis (для кэширования и брокера сообщений)
- Whitenoise (для раздачи статических файлов)

### Frontend
- React 18
- Material-UI
- Redux Toolkit
- React Router
- Axios
- Tailwind CSS

## Архитектура системы

### Архитектура Backend

Backend построен с использованием Django и следует модульной архитектуре:

```
backend/
├── onboardpro/          # Основная конфигурация проекта
│   ├── settings.py     # Настройки проекта
│   └── urls.py        # Конфигурация URL
├── users/              # Модуль управления пользователями
├── onboarding/         # Планы онбординга и задачи
├── core/              # Основной функционал и утилиты
└── api/               # API эндпоинты и сериализаторы
```

#### Ключевые компоненты

1. **Управление пользователями (users/)**
   - Аутентификация и авторизация пользователей
   - Контроль доступа на основе ролей (Сотрудник, Менеджер, HR)
   - Профили пользователей и разрешения

2. **Модуль онбординга (onboarding/)**
   - Управление планами онбординга
   - Создание и отслеживание задач
   - Мониторинг прогресса
   - Управление сроками

3. **API слой (api/)**
   - RESTful эндпоинты
   - JWT аутентификация
   - Сериализация запросов/ответов
   - Версионирование API

4. **Настройки статических файлов**
   - Использование Whitenoise для раздачи статических файлов
   - Конфигурация STATICFILES_DIRS для frontend файлов
   - Поддержка HTML-шаблонов из frontend/src
   - Компрессия и кэширование статических файлов

5. **Маршрутизация**
   - Основные страницы приложения:
     * / - Главная страница
     * /login - Страница входа
     * /dashboard - Панель управления сотрудника
     * /manager-dashboard - Панель управления менеджера
   - API эндпоинты:
     * /api/health - Проверка работоспособности
     * /api/login - Аутентификация
     * /api/refresh - Обновление токена
     * /api/verify-token - Проверка токена
     * /api/test-hr - Тестовый эндпоинт для HR
     * /api/plans/* - Управление планами онбординга
     * /api/tasks/* - Управление задачами

### Архитектура Frontend

Frontend построен на React и следует компонентной архитектуре:

```
frontend/
├── src/
│   ├── components/     # Переиспользуемые UI компоненты
│   ├── pages/         # Компоненты страниц
│   │   ├── login.html        # Страница входа
│   │   ├── dashboard.html    # Панель сотрудника
│   │   └── manager_dashboard.html # Панель менеджера
│   ├── services/      # API сервисы
│   ├── store/         # Redux store и слайсы
│   └── utils/         # Утилиты
├── dist/              # Скомпилированные статические файлы
│   └── output.css     # Скомпилированный CSS (Tailwind)
└── public/            # Публичные статические файлы
```

## Настройка и запуск

### Backend
1. Установка зависимостей:
   ```bash
   cd backend
   python -m pip install -r requirements.txt
   ```

2. Запуск сервера разработки:
   ```bash
   python manage.py runserver
   ```

### Frontend
1. Установка зависимостей:
   ```bash
   cd frontend
   npm install
   ```

2. Компиляция Tailwind CSS:
   ```bash
   npm run build
   ```

## Развертывание

Приложение контейнеризировано с использованием Docker и может быть развернуто с помощью Docker Compose:

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./backend:/app
    environment:
      - DEBUG=1
      - DATABASE_URL=postgres://postgres:postgres@db:5432/onboardpro
    depends_on:
      - db
      - redis

  frontend:
    build: ./frontend
    volumes:
      - ./frontend:/app
      - /app/node_modules
    ports:
      - "3000:3000"

  db:
    image: postgres:13
    environment:
      - POSTGRES_DB=onboardpro
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:6

volumes:
  postgres_data:
```
