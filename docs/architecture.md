# Документация по архитектуре

## Обзор системы

OnboardPro — современное веб-приложение, построенное на Django и React, разработанное для оптимизации процесса онбординга сотрудников. Система следует паттерну микросервисной архитектуры с четким разделением ответственности между backend и frontend компонентами.

## Технологический стек

### Backend
- Django 4.2
- Django REST Framework
- PostgreSQL
- JWT Аутентификация
- Celery (для асинхронных задач)
- Redis (для кэширования и брокера сообщений)

### Frontend
- React 18
- Material-UI
- Redux Toolkit
- React Router
- Axios
- Tailwind CSS

## Архитектура системы

### Архитектура Backend

Backend построен с использованием Django и следует модульной архитектуре:

```
backend/
├── onboardpro/          # Основная конфигурация проекта
├── users/              # Модуль управления пользователями
├── onboarding/         # Планы онбординга и задачи
├── core/              # Основной функционал и утилиты
└── api/               # API эндпоинты и сериализаторы
```

#### Ключевые компоненты

1. **Управление пользователями (users/)**
   - Аутентификация и авторизация пользователей
   - Контроль доступа на основе ролей (Сотрудник, Менеджер, HR)
   - Профили пользователей и разрешения

2. **Модуль онбординга (onboarding/)**
   - Управление планами онбординга
   - Создание и отслеживание задач
   - Мониторинг прогресса
   - Управление сроками

3. **API слой (api/)**
   - RESTful эндпоинты
   - JWT аутентификация
   - Сериализация запросов/ответов
   - Версионирование API

4. **Модели данных (core/models.py)**
   - OnboardingPlan: модель для планов онбординга
   - Task: модель для задач онбординга
   - User: расширенная модель пользователя с ролями

5. **Сериализаторы (core/serializers.py)**
   - OnboardingPlanSerializer: сериализация планов онбординга
   - TaskSerializer: сериализация задач
   - Вложенная сериализация для отображения задач в планах

6. **Представления (core/views.py)**
   - OnboardingPlanViewSet: CRUD операции для планов онбординга
   - TaskViewSet: CRUD операции для задач
   - Фильтрация задач по пользователю для роли employee
   - Специальные представления для ролей manager и hr

7. **Настройки Django (onboardpro/settings.py)**
   - Конфигурация статических файлов
   - Настройки безопасности
   - Конфигурация базы данных
   - Настройки CORS и аутентификации

### Архитектура Frontend

Frontend построен на React и следует компонентной архитектуре:

```
frontend/
├── src/
│   ├── components/     # Переиспользуемые UI компоненты
│   ├── pages/         # Компоненты страниц
│   ├── services/      # API сервисы
│   ├── store/         # Redux store и слайсы
│   └── utils/         # Утилиты
├── dist/              # Скомпилированные статические файлы
│   └── output.css     # Скомпилированный CSS (Tailwind)
└── public/            # Публичные статические файлы
```

#### Ключевые компоненты

1. **Компоненты**
   - Переиспользуемые UI компоненты
   - Компоненты форм
   - Компоненты макета
   - Компоненты навигации

2. **Страницы**

   - Панель управления для HR
   - Планы онбординга
   - Управление задачами

3. **Управление состоянием**

   - Состояние аутентификации
   - Состояние онбординга
   - Состояние задач

4. **API сервисы**

## Потоки данных

1. **Поток аутентификации**
   - Вход пользователя с учетными данными
   - Генерация JWT токена
   - Хранение токена в localStorage
   - Доступ к защищенным маршрутам
   - Перенаправление на соответствующую панель управления в зависимости от роли

2. **Поток онбординга**
   - HR создает план онбординга
   - Задачи назначаются пользователям
   - Пользователи выполняют задачи
   - Отслеживание прогресса и уведомления
   - Менеджеры отслеживают прогресс команды

3. **Поток управления задачами**
   - Создание задач HR/Менеджером
   - Назначение задач пользователям
   - Обновление статуса
   - Отслеживание сроков
   - Фильтрация задач по статусу и пользователю

4. **Поток API взаимодействия**
   - Запросы к API с JWT токеном
   - Сериализация/десериализация данных
   - Обработка ответов и ошибок
   - Кэширование данных на клиенте

5. **Поток статических файлов**
   - Компиляция Tailwind CSS
   - Раздача статических файлов через Django
   - Кэширование статических файлов
   - Оптимизация загрузки ресурсов

## Развертывание

Приложение контейнеризировано с использованием Docker и может быть развернуто с помощью Docker Compose:

```

├── postgres service
└── redis service
```
