# OnboardPro - Система онбординга сотрудников

## Обзор проекта

OnboardPro - это современная система для онбординга и адаптации новых сотрудников в компании. Она позволяет автоматизировать процесс адаптации новичков, отслеживать прогресс задач и аналитику в режиме реального времени.

### Ключевые возможности

- 🔄 **Обновления в реальном времени** - моментальное отображение изменений через WebSocket
- 📊 **Аналитическая панель** - интерактивные графики и метрики эффективности онбординга
- 📝 **Управление задачами** - создание, назначение и отслеживание задач онбординга

- 📆 **Интерактивный календарь** - с функцией drag-and-drop для задач
- 🔔 **Уведомления** - система уведомлений о важных событиях

## Быстрый старт

### Требования

- Docker и Docker Compose
- Node.js 18+ и NPM (для локальной разработки)
- Python 3.11+ (для локальной разработки)

### Установка и запуск

1. Клонировать репозиторий

   ```bash
   git clone https://github.com/your-repo/onboardpro.git
   cd onboardpro
   ```

2. Настроить .env файл с переменными окружения

   ```bash
   cp .env.example .env
   # Отредактировать .env файл с необходимыми настройками
   ```

3. Запустить приложение через Docker

   ```bash
   docker-compose up -d
   ```

4. Открыть приложение в браузере

   ```
   http://localhost:3000
   ```

5. Авторизоваться с демо-пользователем:
   - HR: hr@onboardpro.com / test123
   - Менеджер: manager@onboardpro.com / test123
   - Сотрудник: employee@onboardpro.com / test123

## Архитектура

### Технологический стек

- **Backend**:

  - Python с FastAPI
  - SQLAlchemy ORM
  - SQLite (разработка) / PostgreSQL (продакшн)
  - WebSockets для обновлений в реальном времени
  - Alembic для миграций БД

- **Frontend**:

  - React с функциональными компонентами и хуками
  - TailwindCSS для стилизации
  - Chart.js для визуализации данных
  - FullCalendar для календарного интерфейса

- **Инфраструктура**:
  - Docker и Docker Compose для контейнеризации
  - Nginx для раздачи статики и проксирования запросов

## Ролевая модель

### HR

- Управление пользователями системы
- Создание и редактирование планов адаптации
- Полный доступ к задачам всех сотрудников
- Доступ к аналитике и отчетам
- Управление интеграциями с внешними системами

### Менеджер

- Управление задачами своих сотрудников
- Доступ к профилям сотрудников своего отдела
- Создание и модификация задач
- Обзор прогресса онбординга сотрудников

### Сотрудник

- Просмотр и выполнение назначенных задач
- Обновление статусов задач
- Доступ к личному календарю
- Возможность предоставлять обратную связь

## Развертывание

### Разработка

```bash
# Запуск в режиме разработки
docker-compose -f docker-compose.dev.yml up -d

# Мониторинг логов
docker-compose logs -f

# Остановка контейнеров
docker-compose down
```

### Продакшн

```bash
# Создание продакшн образов
docker-compose -f docker-compose.prod.yml build

# Запуск в продакшн режиме
docker-compose -f docker-compose.prod.yml up -d

# Обновление базы данных
docker exec onboardpro-backend-1 python migration_manager.py
```

## Разработка

### Работа с миграциями

```bash
# Создание новой миграции
docker exec onboardpro-backend-1 alembic revision -m "описание_изменений"

# Применение миграций
docker exec onboardpro-backend-1 python migration_manager.py

# Откат последней миграции
docker exec onboardpro-backend-1 alembic downgrade -1
```

### Внесение изменений в код

При внесении изменений рекомендуется:

4. Запустить тесты и убедиться в их успешном прохождении
