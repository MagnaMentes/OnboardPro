# Часто задаваемые вопросы о проекте OnboardPro

## Общие вопросы

### Что такое OnboardPro?

OnboardPro — это веб-приложение для автоматизации процесса онбординга новых сотрудников в компании. Система позволяет HR-специалистам и менеджерам создавать планы адаптации, назначать задачи новым сотрудникам, отслеживать прогресс и получать аналитику.

### Для кого предназначено приложение?

Приложение разработано для трех основных категорий пользователей:

- **HR-специалисты**: создание и управление планами адаптации, мониторинг прогресса всех сотрудников
- **Менеджеры**: отслеживание и управление задачами своих подчиненных
- **Новые сотрудники**: получение и выполнение задач, отметка о прогрессе

### На каком этапе разработки находится проект?

Проект находится на стадии бета-версии с ограниченным числом пользователей. Основные функции разработаны и стабильны, ведется работа над улучшениями и расширением функциональности.

## Архитектура и технологии

### Какая архитектура используется в проекте?

OnboardPro следует монолитной архитектуре с отдельными слоями для фронтенда и бэкенда, что облегчает начальную разработку и развертывание. В будущем может быть рассмотрен переход к микросервисной архитектуре.

### Какой технологический стек используется?

- **Бэкенд**: FastAPI (Python), SQLAlchemy ORM, SQLite (с планами миграции на PostgreSQL)
- **Фронтенд**: React.js, TailwindCSS, React Router
- **Дополнительные библиотеки**: Chart.js для визуализации данных, FullCalendar для календаря
- **Инфраструктура**: Docker, Docker Compose, Nginx

### Как организована аутентификация?

Аутентификация реализована с использованием JWT (JSON Web Tokens). Пользователи аутентифицируются через эндпоинт `/login`, получают токен, который затем используется для доступа к защищенным ресурсам.

## Функциональность

### Какие основные возможности предоставляет OnboardPro?

- Система аутентификации и ролевой доступ (HR, менеджер, сотрудник)
- Управление пользователями (создание, редактирование, блокировка)
- Создание и редактирование планов адаптации
- Управление задачами онбординга с отслеживанием статусов
- Обратная связь между пользователями
- Аналитика и статистика онбординга
- Интеграции с внешними сервисами

### Какие внешние интеграции поддерживаются?

- Telegram Bot: отправка уведомлений о задачах и их статусах
- Google Calendar: синхронизация задач с календарем пользователя
- Workable: импорт данных о новых сотрудниках из HR-системы

### Как реализовано реальное время в приложении?

Для обновлений в реальном времени используется WebSocket. При изменении статуса задачи или других важных событиях сервер отправляет уведомление подключенным клиентам без необходимости обновления страницы.

## Разработка и эксплуатация

### Как запустить проект локально для разработки?

Проект можно запустить с помощью Docker Compose:

```bash
git clone https://github.com/MagnaMentes/OnboardPro.git
cd OnboardPro
cp backend/.env.example backend/.env
docker-compose up -d
```

### Как запустить тесты?

Модульные тесты бэкенда:

```bash
cd backend
pytest tests/test_api.py -v
```

Нагрузочное тестирование:

```bash
./run_load_test.sh
python backend/analyze_load_test.py
```

### Какие известные проблемы есть в текущей реализации?

- Неоптимальные запросы к базе данных при большом количестве записей
- SQLite имеет ограничения при конкурентном доступе
- Проблемы с синхронизацией состояний между фронтендом и бэкендом
- Некорректное отображение времени в зависимости от часового пояса
- Проблемы с адаптивностью на мобильных устройствах для некоторых компонентов
- Отсутствие механизма повторных попыток при сбое внешних сервисов

## Ближайшие планы развития

### Какие улучшения планируются в ближайшее время?

1. Миграция с SQLite на PostgreSQL для улучшения производительности и надежности
2. Улучшение механизмов безопасности (двухфакторная аутентификация, улучшение политик паролей)
3. Оптимизация производительности фронтенда
4. Разработка API для мобильного приложения
5. Добавление шаблонов планов для быстрого создания типовых задач

### Какие долгосрочные планы развития?

- Расширение аналитического модуля с более сложными отчетами и визуализациями
- Интеграция с дополнительными внешними сервисами
- Разработка мобильного приложения
- Масштабирование инфраструктуры для поддержки крупных корпоративных клиентов
