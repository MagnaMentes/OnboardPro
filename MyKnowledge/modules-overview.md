# Обзор модулей OnboardPro

## Бэкенд (FastAPI)

### Основные модули

#### `main.py`

- **Описание**: Основной файл с определением API эндпоинтов приложения
- **Функциональность**:
  - Настройка FastAPI приложения и CORS
  - Определение всех эндпоинтов REST API (логин, пользователи, задачи, планы, обратная связь, аналитика)
  - Обработчики WebSocket для обновлений в реальном времени
  - Система кэширования аналитических данных в памяти
  - Валидация входных данных через Pydantic-модели

#### `models.py`

- **Описание**: Определение ORM-моделей SQLAlchemy
- **Основные модели**:
  - `User`: Пользователи системы с разными ролями
  - `OnboardingPlan`: Планы адаптации
  - `Task`: Задачи в рамках онбординга
  - `Feedback`: Обратная связь между пользователями
  - `Analytics`: Аналитические данные и метрики
- **Особенности**: Установлены отношения между моделями для эффективной работы ORM

#### `auth.py`

- **Описание**: Модуль аутентификации и авторизации
- **Функциональность**:
  - JWT-аутентификация
  - Хеширование и проверка паролей
  - Функции для получения текущего пользователя из запроса
  - Зависимости FastAPI для защиты эндпоинтов

#### `database.py`

- **Описание**: Настройка подключения к базе данных
- **Функциональность**:
  - Конфигурация SQLAlchemy
  - Создание и управление сессиями базы данных
  - Инициализация соединения с SQLite

#### `integrations.py`

- **Описание**: Интеграции с внешними сервисами
- **Поддерживаемые интеграции**:
  - Telegram Bot для отправки уведомлений
  - Google Calendar для синхронизации задач
  - Workable для импорта данных о сотрудниках

#### `websocket_manager.py`

- **Описание**: Управление WebSocket соединениями
- **Функциональность**:
  - Установка и закрытие WebSocket-соединений
  - Отправка обновлений о статусе задач в реальном времени
  - Трансляция обновлений аналитики для HR-пользователей
  - Обработка различных типов сообщений от клиентов
- **Важные особенности**:
  - Многоуровневая обработка исключений для надежной работы в Docker
  - Строгая последовательность операций: сначала accept(), затем работа с соединением
  - Поддержка авторизации через JWT-токены
  - Раздельные обработчики для различных типов пользователей
  - Механизм хранения и обновления соединений по user_id

### CORS и безопасность

#### CORS middleware

- **Расположение**: Настраивается в `main.py`
- **Особенности настройки**:
  - Настройка CORS с явным указанием разрешенных источников
  - Включает несколько origin-адресов для различных сред (локальной, Docker)
  - Поддержка credentials для авторизованных запросов
  - Прокидывание заголовков авторизации
- **Важные аспекты**:
  - CORS middleware должен идти после обработчика исключений
  - Для правильной работы с ошибками используется специальный middleware, который добавляет CORS-заголовки к ответам с ошибками

#### Middleware для обработки исключений

- **Расположение**: Определен в `main.py` перед CORS middleware
- **Функциональность**:
  - Перехват всех исключений до их обработки CORS middleware
  - Добавление CORS-заголовков к ответам с ошибками
  - Позволяет браузеру корректно обрабатывать ответы с ошибками, не блокируя их из-за отсутствия CORS-заголовков
  - Логирование ошибок для диагностики

### API модули

#### `api/analytics.py`

- **Описание**: API для аналитических данных
- **Функциональность**:

  - Механизм кэширования для повышения производительности
  - Инвалидация кэша при изменении данных
- **Ключевые компоненты**:
  - Система версионирования данных аналитики
  - Механизм кэширования с поддержкой времени жизни кэша
  - Функции для получения и установки кэшированных данных
  - Генерация ключей кэша на основе параметров запроса

## Фронтенд (React)

### Основные компоненты

#### `src/App.js`

- **Описание**: Основной компонент приложения
- **Функциональность**:
  - Настройка маршрутизации (React Router)
  - Определение публичных и защищенных маршрутов
  - Проверка ролей для доступа к разным частям приложения

#### `src/components/`

- **Описание**: Переиспользуемые компоненты
- **Подкатегории**:
  - `common/`: Общие компоненты (кнопки, поля ввода, модальные окна, таблицы, карточки)
  - `layout/`: Компоненты макета (Header, Footer, Sidebar)
  - `specific/`: Специфичные компоненты (формы задач и планов, профили, карточки задач, аналитика)

#### `src/pages/`

- **Описание**: Компоненты страниц приложения
- **Основные страницы**:
  - `Login.jsx`: Страница входа в систему
  - `Dashboard.jsx`: Дашборд сотрудника с задачами и прогрессом
  - `ManagerDashboard.jsx`: Дашборд менеджера для управления задачами команды
  - `HRDashboard.jsx`: Дашборд HR с аналитикой
  - `Profiles.jsx`: Управление профилями пользователей
  - `Feedback.jsx`: Работа с обратной связью
  - `Integrations.jsx`: Настройка интеграций

#### `src/context/`

- **Описание**: React Context-провайдеры для глобального состояния
- **Основные контексты**: Пользователь, темы оформления, состояние UI

#### `src/hooks/`

- **Описание**: Пользовательские React-хуки
- **Примеры хуков**: usePageTitle, useApi, useAuth

#### `src/utils/`

- **Описание**: Утилиты и хелперы
- **Включает**: форматирование данных, валидация, вспомогательные функции

#### `src/config/`

- **Описание**: Конфигурационные файлы
- **Включает**: настройки API, константы, параметры окружения
