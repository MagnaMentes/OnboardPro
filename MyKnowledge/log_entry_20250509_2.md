# 9 мая 2025 (дополнение)

## Исправление проблемы с сохранением выбранного отдела в карточке пользователя

Сегодня была обнаружена и исправлена проблема с функционалом сохранения выбранного отдела при редактировании карточки пользователя.

### Описание проблемы

1. При редактировании карточки пользователя через форму в `Profiles.jsx` пользователь выбирает отдел из выпадающего списка отделов, который заполняется данными из `/api/departments`.
2. При этом в базе данных есть два поля для хранения информации об отделе пользователя:
   - Строковое поле `department` - название отдела
   - Числовое поле `department_id` - внешний ключ к таблице `departments`
3. При сохранении формы в API передавался только `department_id`, но на стороне API не было обработки этого поля, и оно не сохранялось.
4. В итоге в карточке пользователя отображался статус "Не указан" для отдела, хотя отдел был выбран.

### Внесенные изменения

#### Backend (main.py)

1. Обновлены модели `UserCreate` и `UserUpdate` для поддержки поля `department_id`:

   ```python
   class UserUpdate(BaseModel):
       email: str
       role: str
       department: str | None = None
       department_id: int | None = None  # Добавлено это поле
       first_name: str | None = None
       # ... другие поля
   ```

2. Обновлен эндпоинт `update_user` для корректной обработки поля `department_id`:

   ```python
   if user_data.department_id:
       # Если указан ID отдела, проверяем его существование и берём название из базы
       department = db.query(models.Department).filter(models.Department.id == user_data.department_id).first()
       if department:
           db_user.department_id = user_data.department_id
           db_user.department = department.name
   ```

3. Обновлен эндпоинт `create_user` для корректной обработки поля `department_id` при создании нового пользователя.

#### Frontend (Profiles.jsx)

1. Обновлена функция `handleChange` для обработки выбора отдела:

   ```javascript
   if (name === "department_id") {
     if (value) {
       const selectedDepartment = departments.find(
         (dept) => dept.id.toString() === value
       );
       if (selectedDepartment) {
         setFormData({
           ...formData,
           department_id: value,
           department: selectedDepartment.name,
         });
         return;
       }
     }
   }
   ```

2. Обновлены функции `openCreateModal` и `openEditModal` для правильной инициализации полей формы с учетом `department_id`.

### Результат

Теперь при редактировании карточки пользователя корректно сохраняются и `department_id`, и `department`, что позволяет:

1. Правильно отображать название отдела в карточке пользователя
2. Сохранять связь с таблицей отделов через внешний ключ
3. Обеспечить целостность данных в базе

Такой подход обеспечивает как удобство пользовательского интерфейса, так и корректную работу с данными на уровне базы.
