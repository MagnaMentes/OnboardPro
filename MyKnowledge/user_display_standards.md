# Стандарты форматирования данных пользователей

## Введение

В данной документации описаны стандарты отображения имен пользователей в интерфейсе приложения OnboardPro. Следование этим стандартам обеспечивает единообразие пользовательского опыта во всем приложении.

## Проблема

В предыдущих версиях приложения имена пользователей отображались непоследовательно:

- В некоторых компонентах отображались email пользователей
- В других компонентах имя и фамилия отображались с использованием различных подходов
- Отсутствовала унифицированная обработка случаев, когда имя/фамилия не указаны

## Решение

Для обеспечения единообразия и повышения качества UX разработана библиотека утилитарных функций `userUtils.js`.

### Основные функции

1. `formatUserDisplayName(user)` - форматирует имя пользователя для отображения
2. `getUserDisplayNameById(userId, users)` - находит и форматирует имя пользователя по ID
3. `getUserInitials(user)` - получает инициалы пользователя для аватаров
4. `hasFullName(user)` - проверяет заполненность имени и фамилии

### Примеры использования

```jsx
import {
  formatUserDisplayName,
  getUserDisplayNameById,
} from "../utils/userUtils";

// Отображение имени пользователя
const UserCard = ({ user }) => (
  <div className="user-card">
    <h3>{formatUserDisplayName(user)}</h3>
    {/* Другие поля */}
  </div>
);

// Отображение исполнителя задачи по ID
const TaskItem = ({ task, users }) => (
  <div className="task-item">
    <p>Назначена: {getUserDisplayNameById(task.user_id, users)}</p>
    {/* Другие поля */}
  </div>
);
```

### Указания по использованию

1. Всегда используйте `formatUserDisplayName` вместо прямого доступа к полям пользователя
2. Для компонентов, где доступен только ID пользователя, используйте `getUserDisplayNameById`
3. При отображении аватаров используйте `getUserInitials` для заполнителя
4. Не создавайте собственных функций форматирования имен - используйте существующие утилиты

## Рекомендации по дальнейшему развитию

1. Рассмотреть возможность добавления поля `display_name` на бэкенде:

   - Вычисляемое поле, которое автоматически формируется из first_name и last_name
   - Обновляется при изменении данных пользователя
   - Упрощает получение данных для отображения

2. Обеспечить локализацию форматирования имен для международной поддержки

## Список компонентов, где применён стандартизированный подход

- TaskModal.jsx - отображение назначенного исполнителя с использованием `formatUserDisplayName`
- ManagerDashboard.jsx - заменена локальная функция `getUserDisplayNameById` на импорт из модуля `userUtils.js`
- DepartmentForm.jsx - отображение менеджера отдела с использованием `formatUserDisplayName`
- PlanForm.jsx - выбор пользователя для плана адаптации (изначально правильный подход)
- TaskForm.jsx - выбор исполнителя задачи (изначально правильный подход)

## Внедрение утилит в существующий код

Во всех компонентах, где происходит отображение данных пользователей, осуществлена следующая стандартизация:

1. Импортированы необходимые функции:

   ```jsx
   import {
     formatUserDisplayName,
     getUserDisplayNameById,
   } from "../utils/userUtils";
   ```

2. Замена прямых обращений к полям пользователя на вызов утилитарной функции:

   ```jsx
   // До
   {
     user.first_name && user.last_name
       ? `${user.first_name} ${user.last_name}`
       : user.email;
   }

   // После
   {
     formatUserDisplayName(user);
   }
   ```

3. Для компонентов, где используется только ID пользователя:

   ```jsx
   // До
   const user = users.find((u) => u.id === userId);
   return user
     ? user.first_name && user.last_name
       ? `${user.first_name} ${user.last_name}`
       : user.email
     : "";

   // После
   return getUserDisplayNameById(userId, users);
   ```

## Заключение

Унификация отображения имен пользователей улучшает читаемость интерфейса и помогает пользователям быстрее идентифицировать сотрудников. Систематический подход обеспечивает последовательность во всем приложении и упрощает поддержку кода в будущем.

Рекомендуется включить проверку использования этих утилит в процесс code review для новых компонентов и при рефакторинге существующих.
